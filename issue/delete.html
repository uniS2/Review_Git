issue1. 로컬 저장소 삭제

  1. 우선 이럴 경우를 대비해 항상 commit 후 반드시 push하는 습관을 들여야한다. 그러지 않았을 경우..? 간단하다 밤을 ㅅ...ㅐ...ㅈ...
  2. 항상 push 하였다는 가정하에 방법은 간단하다.
    새로운 폴더 만들기 > git init 명령 > 원격 저장소와 연결 > 할렐루야!
  3. push를 하지 못했다, 원격 저장소도 없어보인다.
    이런 경우는 지옥문이 열렸다고 봐야한다. 우선은 일말의 희망을 가지고 'gitHub의 설정>저장소>삭제한저장소'를 뒤진다. 여기에도 없다면 당장 컴퓨터 삭제 복원 프로그램을 깔아라. 그래도 안된다? 행운을 빕니다...★
  4. 결론은 뭐? commit-push 생활화 하세요!
